Что мешает новой фиче

Архитектура сейчас разделена по интеграциям, а не по фичам: internal/telegram, internal/discord, internal/storage. Для масштабирования это неудобно.
Нет абстракции "feature/module", которая могла бы подписываться на события и решать: когда слать мем, gif, sticker, sound.
Нет хранилища медиа-ресурсов, тегов, триггеров, cooldown и политики отправки.
Нет входящего Telegram message/event слоя для реакции на сообщения в беседе, если ты хочешь слать мемы "по ситуации", а не только по Discord-событиям.
Как я бы разнёс по папкам

cmd/bot — только bootstrap.
internal/platform/telegram — низкоуровневый Telegram client/adapters.
internal/platform/discord — низкоуровневый Discord client/adapters.
internal/platform/storage — SQLite и репозитории.
internal/features/presence — текущая логика voice-статусов и deep-link flow.
internal/features/media — отправка мемов, gif, sticker, sound.
internal/features/media/catalog — каталог медиа, теги, поиск, cooldown.
internal/features/media/triggers — правила "когда отправлять".
internal/features/media/providers — Telegram sticker/GIF/@sounds adapters.
internal/app — оркестрация модулей, DI, event bus/interfaces.
Практичная реализация media-фичи

Начать без нейросети: curated media packs + теги + правила сработки. Для ноутбука i5 это самый надёжный путь.
Хранить в SQLite или JSON/YAML:
media_items
media_tags
media_rules
media_cooldowns
Поддержать типы:
sticker_file_id
animation_file_id
gif_url или Telegram animation
audio/voice
text+media
Telegram лучше отправлять по file_id, если медиа уже однажды загружено в бота: это быстрее и дешевле.
Логику срабатывания делать через rule engine:
по ключевым словам
по regex
по Discord voice event
по Telegram reply/mention
по random chance
по cooldown на чат/пользователя
Про YOLO

Для задачи "отправлять мемчики, @sounds, гифки, стикеры" YOLO обычно не нужен.
YOLO полезен, если ты хочешь распознавать, что на картинке: лицо, объект, сцена, NSFW-модерация через доп. пайплайн.
На простом i5 я бы не ставил YOLO в критический runtime-path бота.
Если очень нужен CV, то только маленькая CPU-модель уровня yolov8n/ONNX и лучше асинхронно, а не на каждый апдейт.
Для первого релиза выгоднее: правила + каталог контента + ручные теги.
